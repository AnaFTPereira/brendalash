Você é um senior full-stack + mobile engineer em uma multinacional. Gere um repositório Git completo, pronto para GitHub, que contenha:

Site institucional (Next.js, TypeScript, Tailwind CSS), totalmente responsivo, acessível, SEO otimizado e pronto para deploy no GitHub Pages ou Vercel.

Aplicativo mobile (Expo, React Native, TypeScript, NativeWind) para Android/iOS, com mesma identidade visual, focado em UX mobile-first, pronto para build via Expo EAS.
Ambos devem compartilhar conteúdo, branding e lógica em um monorepo organizado.

Objetivo

Web responsivo (desktop/tablet/mobile) + App mobile.

Home (profissional): bio, serviços, galeria, depoimentos, agendamento, contato.

Página/Tela Evento: hero, descrição, programação, instrutor, inscrição (RSVP + exemplo Stripe), local (mapa), galeria, FAQ.

Design unificado → cores, tipografia, ícones.

Internacionalização (pt/en).

Documentação completa no README.md.

CI/CD para Web (GitHub Pages/Vercel) e Mobile (Expo EAS).

Testes (unitários e e2e).

Funcionalidades comuns (Web + Mobile)

Home: hero, bio, serviços (cards), portfólio (galeria com zoom), depoimentos, CTA de agendamento (WhatsApp/formulário).

Evento: hero com título, data, local, descrição, programação, instrutor, inscrição (formulário RSVP + Stripe demo), mapa, FAQ, galeria.

Contato: formulário + WhatsApp + redes sociais.

Agendamento: formulário validado + link direto para WhatsApp.

Internacionalização: packages/content → pt.json e en.json.

Responsividade & UX

Web:

Grid flexível (Tailwind).

Breakpoints otimizados para mobile-first (sm, md, lg, xl).

Galeria responsiva (carrossel + modal).

Mobile (Expo):

Navegação → bottom tab (Home | Serviços | Evento | Contato).

Carrosséis touch-friendly.

Mapa interativo (Expo Location + MapView).

Animações suaves (Reanimated + Moti).

Integrações

WhatsApp: link universal (wa.me).

Formulários: client-side + API (SendGrid placeholder).

Stripe: demo de checkout para inscrição no evento.

Analytics: GA4 no Web (via consentimento), Expo Analytics no Mobile.

Mapa: Google Maps iframe (Web) e react-native-maps (Mobile).

Acessibilidade

Web: semântica HTML, ARIA, contraste ≥ 4.5:1, navegação por teclado.

Mobile: suporte a screen readers, labels acessíveis, foco visual em inputs.
